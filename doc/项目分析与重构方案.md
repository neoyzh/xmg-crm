## 一、项目功能点与流程结构

### 1.1 技术栈

| 类别 | 选型 |
|------|------|
| 构建工具 | Maven（WAR 包） |
| 后端框架 | Spring MVC 4.2.3 + Spring 4.2.3 + MyBatis 3.2.3 |
| 数据库 | MySQL 8，Druid 1.0.14 连接池 |
| 前端 | JSP + jQuery EasyUI + FullCalendar v1.x |
| 搜索 | Lucene 4.10.4 + IK Analyzer 中文分词 |
| 调度 | Quartz |
| WebService | Apache CXF 3.0.3（SOAP） |
| Excel | jxl（JExcelAPI） |
| 其他 | jxl、Hutool、json-lib |
| 部署 | tomcat7-maven-plugin，端口 9527 |

### 1.2 业务模块（18 个）

| 模块 | 核心流程 |
|------|---------|
| 员工管理 | 登录（MD5 校验）→ RBAC 权限（角色-权限-菜单三级）→ Session 存储 |
| 部门管理 | 树形组织架构，支持父子部门 |
| 角色/权限 | 角色-权限多对多，员工-角色多对多 |
| 客户管理 | 潜在客户(0) → 开发中 → 正式客户(1) / 开发失败(-1) / 流失(-2) / 资源池(2) |
| 客户移交 | 专员间共享 / 主管分配 → 生成移交记录 |
| 资源池 | 进入资源池 → 认领 |
| 开发计划 | 潜在客户开发计划(type=0) + 正式客户跟进计划(type=1) |
| 合同管理 | 订单 → 合同生成 → 审核流程（状态递进） → 生成保修单（1 年） |
| 订单管理 | 录入订单 → 审核 → 生成合同 |
| 保修管理 | 合同 → 保修单 → 维修记录（最多 3 次免费） → 延保（+1 年） |
| 考勤模块 | 签到 → 签退 → 考勤统计（正常/迟到/早退）→ Excel 导出 |
| 请假管理 | 提交 → 审核（HR） |
| 薪资管理 | Excel 导入 → 按员工/月份查询 |
| 任务管理 | 分配任务 → 7 天内完成/失败 → 状态流转 |
| 报表中心 | 客户/合同图表（折线/饼图/柱状图），按年/月/季度分组 |
| 知识库 | Lucene + IK 分词 → 全文检索 |
| 线索管理 | 爬虫数据 → Lucene 索引 → Quartz 定时同步（每日 23:59） |
| 网盘 | 文件夹/文件上传下载，层级管理 |
| 日历 | FullCalendar 事件拖拽 |
| 数据字典 | 目录 + 条目二级管理，禁用/启用 |

### 1.3 请求处理流程

```
HTTP 请求
  ↓
CharacterEncodingFilter（UTF-8）
  ↓
DispatcherServlet → application-mvc.xml
  ↓
LoginInterceptor.preHandle()
  ├─ UserContext.set(request)          // ThreadLocal 存请求
  ├─ 校验 Session 登录态                // 未登录 → /login.jsp
  └─ PermissionUtils.checkPermission() // 拼接 ClassName:methodName 校验权限
  ↓
@Controller 方法
  ↓
@Service（事务：读 SUPPORTS / 写 REQUIRED）
  ↓
@Mapper（MyBatis XML 或注解 SQL）
  ↓
AjaxResult / PageResult（JSON 响应）
```

### 1.4 横切关注点

- **认证**：基于 Session，登录时存储 Employee + 权限列表 + 菜单树
- **授权**：表达式匹配（`ClassName:methodName`），超级管理员绕过
- **日志**：AOP 后置通知拦截所有 `*Service.*(..)`，记录到 log 表
- **事务**：Spring 声明式事务，read-only 用 SUPPORTS
- **搜索**：Lucene 索引 clew / know 表内容，Quartz 每日同步
- **文件上传**：MultipartFile → webapp/upload/ 与 netdisk/

## 三、重构方案（分批执行）

### 第一批：Bug 修复（当前执行）

1. **B1** — 重写 `CustomerChartMapper.xml` 的 `queryByCondition`，删除断裂的半截 SELECT，整合为完整查询，对齐 VO 字段映射
2. **B2** — 删除 `TaskController` 中 `task_complateAll` / `task_loseAll` 两个编译失败方法
3. **B3** — 修正 `CustomerChartVoMap` 的空 column，或直接移除未使用的 resultMap

### 第二批：安全 + 代码质量（后续）

- 引入 BCrypt / SHA-256 + salt 替换 MD5
- 移除硬编码密钥
- 统一异常处理（`@ControllerAdvice`）
- 清理 debug 输出与死代码
- 移动 `CustomerMapper.xml` 到 `resources`
- 替换 `sun.misc.ProxyGenerator`

### 第三批：架构优化（后续）

- Service 层收拢业务逻辑
- 删除装饰性 `@RequiredPermission`
- 合并重复的 transfer / shareOrTransfer
- 签到查询用 SQL 条件过滤，避免全量加载
- 统一 AjaxResult 使用方式
